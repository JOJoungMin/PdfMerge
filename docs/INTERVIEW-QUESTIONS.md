## 4. 1분 자기소개 (학습·철학·스토리 버전)

output-1 이력서의 **확장성·품질·검증**을 살리면서, 경험 → 깨달음 → 철학으로 이어지는 스토리.

---

**안녕하세요, 신입 백엔드 개발자 조정민입니다.**

인턴 기간 동안 팀 프로젝트 한 개와 개인 프로젝트 한 개를 개발·배포했습니다.

팀 프로젝트(Lyric Type)에서는 Python·Flask로 백엔드 API와 인프라를 담당했고, 모킹 서버와 API 명세서로 팀원 개발 지연을 막았으며, Locust·pytest로 병목을 찾아 Redis로 성능을 개선했습니다. 이 과정에서 **"테스트와 측정 없이는 개선도 없다"**는 걸 배웠습니다.

개인 프로젝트(PDF-Utils)에서는 NestJS로 PDF 유틸 서비스를 만들었고, Controller와 Service를 분리하고 E2E 테스트 통과 후에만 배포되도록 구성했습니다. 처음에는 기능만 빠르게 만들었는데, 나중에 수정·추가가 어려워지면서 **"처음부터 확장 가능한 구조를 잡아야 한다"**는 걸 느꼈습니다.

그래서 **확장성과 품질·검증을 중요하게 보는 개발**을 하고 싶어졌고, 그런 관점이 필요한 데이타솔루션에 지원했습니다.

---

**구성 포인트**

| 구간 | 내용 | 연결 |
|------|------|------|
| 팀 프로젝트 | Locust·pytest·Redis | → "테스트·측정 없이는 개선 없다" (학습 가능성) |
| 개인 프로젝트 | FSD·레이어드·E2E | → "처음부터 확장 가능한 구조가 필요하다" (개발 철학) |
| 마무리 | 확장성·품질·검증 | → 데이타솔루션 지원 이유로 연결 |

**말하기 팁:** 1분이면 위 본문 그대로, 30초면 팀/개인 각 1~2문장 + 마무리 1문장으로 압축.

## 5. EC2 도커·스케줄링 (가상화 자동화·성능 안정화)

**공고 키워드**: 가상화 자동화, 성능 안정화, 도커 컨테이너 관리, 스케줄링

### 구현 내용 (deploy/ 폴더)

| 항목 | 설명 |
|------|------|
| **Docker Compose** | 백엔드 replica 2개 + Nginx 부하분산 |
| **Self-healing** | `restart: unless-stopped` — 컨테이너 죽으면 자동 재시작 |
| **스케줄링** | `scheduler.sh` — 헬스체크, 미사용 이미지 정리 (cron) |

### 면접에서 말할 스토리

> "비용 때문에 EC2를 늘리지 않고, 한 인스턴스 안에서 Docker Compose로 백엔드를 replica 2개 띄워 트래픽을 분산했습니다. Nginx로 부하분산하고, `restart: unless-stopped`로 컨테이너가 죽으면 자동 재시작되게 했고, cron으로 매일 헬스체크와 미사용 이미지 정리를 돌려 스케줄링 관리를 했습니다."

### 예상 질문 → 답변

| 질문 | 답변 포인트 |
|------|-------------|
| 유저가 100만 명으로 늘어나면? | "당장은 한 EC2 안에서 replica로 처리량을 늘렸고, 진짜로 부족하면 오토 스케일링 그룹·K8s로 확장할 계획입니다." |
| Self-healing이 뭔가요? | "컨테이너가 죽으면 Docker가 `restart: unless-stopped`로 자동 재시작합니다. SSH로 들어가 수동 재시작하던 걸 대신 해줍니다." |
| 스케줄링은? | "cron으로 매일 새벽 헬스체크하고, 실패 시 compose 재기동. 미사용 이미지·컨테이너 prune으로 디스크 관리합니다." |

## 6. 활용 팁

- **공고**: 선임은 Java/Spring 또는 C#/.NET, 아키텍처·레거시·테스트 경험 위주로 준비. 수석은 k8s·가상화·마이그레이션·기술 의사결정 스토리 정리.
- **후기**: 이력서/자소서 한 줄 한 줄 설명·숫자·결과 준비. 손코딩은 기본 문법·반복문·배열·간단 알고리즘(정렬·검색·홀수 출력 등) 복습.

- **이력서**: PDF-Utils(풀스택·FSD·E2E·배포), Lyric Type/TypeSomething(백엔드·CI/CD·Redis·성능), Secui 인턴(API TC·테스트 자동화), YouSync(데이터 전처리·팀리드)는 각 1~2분 스토리로 정리해 두면 유리함.
