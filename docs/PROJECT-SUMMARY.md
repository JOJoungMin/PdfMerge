# Project

**[PDF-Utils] PDF 유틸리티 웹 서비스 개발**

| 구분 | 내용 |
|------|------|
| **기간** | 2025.08 ~ 진행 중 |
| **기술 스택** | Next.js, NestJS, TypeScript, Playwright, GitHub Actions, Vercel, ECR, Docker, EC2, Ghostscript, pdf-lib, sharp, tesseract.js |
| **팀** | 1인 (개인 프로젝트) |
| **서비스** | PDF 병합·분리·압축·변환·회전·블라인드·페이지번호·이미지→PDF 등 8개 기능을 제공하는 올인원 웹 툴. 회원가입 없이 브라우저에서 바로 사용. |

## 문제 / 해결 / 결과

- **배포 후 검증 문제**  
  EC2에 프론트·백을 함께 배포하면, 이미 올라간 상태에서 E2E를 돌게 되어 **문제 있는 버전이 프로덕션에 노출될 수 있음**.  
  → 프론트는 Vercel, 백엔드만 EC2 스테이징으로 두고, GitHub Actions에서 EC2 스테이징 대상으로 Playwright E2E 실행 후 **모든 테스트 통과 시에만** Vercel 프로덕션 배포 트리거.  
  → **배포 전 검증**으로 품질 보장.

- **백엔드 배포 방식**  
  docker-compose 대신 **docker run 단일 컨테이너**로 EC2에 백엔드만 배포하고, DB 등 환경 변수를 직접 주입해 구성 단순화 및 운영 부담 감소.

- **E2E 유지보수·중복**  
  비슷한 페이지(분리·압축·회전·블라인드·페이지번호)마다 동일한 TC를 복붙하면 수정 시 누락·중복 발생.  
  → **페이지 배열 기반 파라미터 테스트**로 “홈 갔다 오면 빈 화면”, “파일 올리면 미리보기” 등을 한 TC로 여러 페이지에 적용.  
  → 시나리오 수 유지하면서 코드 중복 감소 및 수정 포인트 일원화.

- **업로드 후 UX·상태**  
  파일 업로드 직후 화면 전환이 늦고, 다른 페이지 갔다 오면 이전 파일이 남아 혼란.  
  → 파일 추가 시 **상태를 먼저 반영**하고 미리보기는 비동기로 채우도록 해 **즉시 레이아웃 전환** + “PDF 빌드중” 오버레이 표시. 페이지 이탈 시 **unmount에서 store reset**으로 재진입 시 빈 화면.  
  → 체감 반응 속도 개선 및 상태 일관성 확보.

- **API·에러 맥락 추적**  
  에러 발생 시 어떤 요청·경로에서 났는지 추적하기 어려움.  
  → **traceId(요청 ID)** 부여, **LoggingInterceptor**로 요청/응답·에러를 **request_logs 테이블**에 저장.  
  → 로그·DB로 에러 시점의 맥락 파악 가능.

- **기능별 API·연속 작업 UX**  
  병합·압축·변환 등 기능이 흩어지면 “결과를 다른 기능으로 이어 쓰기”가 불편.  
  → 기능마다 **전용 REST API** 설계하고, 처리 결과를 **전달 사이드바**로 넘겨 “다른 기능 사용하기”에서 병합→압축→변환처럼 **연속 작업** 가능하도록 API·전역 상태(transfer sidebar) 연동.  
  → 단일 기능을 넘어 **워크플로** 지원.

- **아키텍처·확장성**  
  페이지·기능이 늘어날 때 구조가 흐트러지기 쉬움.  
  → 프론트는 **FSD(app / widgets / features / shared)** 로 기능 단위 분리, 백엔드는 **NestJS 모듈(Controller–Service)** 및 pdf / admin / database 등 **도메인 단위 구성**.  
  → 개발·협업·기능 추가 시 확장성·유지보수성 확보.

- **PDF 처리·미리보기**  
  서버에서 PDF 미리보기·압축·변환을 안정적으로 처리해야 함.  
  → **Ghostscript**(미리보기·압축), **Poppler**(메타정보), **pdf-lib**(병합·분리·페이지 조작), **sharp**(이미지), **tesseract.js**(OCR·블라인드) 조합으로 기능 구현. Docker 이미지에 gs·poppler 포함.  
  → 8개 PDF 기능을 하나의 백엔드에서 제공.

- **다운로드 남용 방지**  
  비로그인 사용자 무제한 다운로드 시 리소스·비용 부담.  
  → **일일 다운로드 횟수 제한**(로컬 저장 기반 카운트)으로 익명 사용자 사용량 제한.  
  → 서버 부하·비용 완화.
